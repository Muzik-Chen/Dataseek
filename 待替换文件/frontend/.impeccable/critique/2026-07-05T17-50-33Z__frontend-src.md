## Overall Impression

A competent Vue 3 + Element Plus scaffold with unexpectedly strong engineering fundamentals (token system, state coverage, responsive architecture). But the design layer never graduated beyond "make it work with Element Plus." The blue primary color, emoji iconography, and interchangeable card templates create a dissonance with the PRODUCT.md's vivid brand brief: this reads as a well-organized CRUD app, not a Chaoshan culture platform. The single biggest opportunity: flip `--primary` from corporate blue to 英歌舞朱红, then systematically replace generic patterns with culturally-specific alternatives. The engineering is solid; the design needs the same level of cultural commitment.

## What's Working

1. **Design token system** (`global.css:1-57`) — Well-structured OKLCH tokens, 4pt spacing scale, fixed rem type scale, semantic z-index/shadow layers, and accessibility-conscious features (prefers-reduced-motion, focus-visible). Above-average engineering for this project stage.

2. **State coverage discipline** — Every data-fetching view handles loading (LoadingSkeleton), empty (EmptyState), and error (el-alert) states. The LoadingSkeleton component has card/list/banner variants with a tasteful shimmer animation. Present in 7 of 8 main views.

3. **Responsive architecture** — Three breakpoints (1024px, 768px, 640px) with adapted grids, mobile drawer nav, full-width hero images on small screens. Mobile-first intent from Design Principle 5 is partially realized.

## Priority Issues

### [P0] Primary color contradicts brand personality
**Why it matters**: PRODUCT.md explicitly rejects "蓝白配色" and defines the brand as 鲜活/热烈/有冲击力 with 英歌舞朱红 as the primary cultural reference. `--primary: oklch(0.52 0.16 248)` is a cool corporate blue that makes every page feel like a SaaS dashboard. This propagates through navigation, buttons, links, focus rings, and active states across all 8 modules.
**Fix**: Replace `--primary` with warm vermillion red (`oklch(0.55 0.22 28)`), make `--accent` a gold/amber from 工夫茶. Warm the surface: `oklch(0.97 0.01 60)` instead of cool `oklch(0.96 0.005 248)`.
**File**: `src/assets/styles/global.css`
**Suggested command**: `/impeccable colorize frontend/src`

### [P0] Search triggers full page reload instead of client-side navigation
**Why it matters**: `window.location.href` on search (AppHeader.vue:123) destroys all SPA state, cancels requests, resets Pinia stores, and creates a visible full-page flash. For mobile users on slow connections this is jarring. The most prominent utility control behaves fundamentally differently from every other navigation element.
**Fix**: Replace with `router.push({ path: '/search', query: { keyword: keyword.trim() } })`.
**File**: `src/components/layout/AppHeader.vue`
**Suggested command**: `/impeccable polish src/components/layout/AppHeader.vue`

### [P1] Dashboard all data fetches fail silently
**Why it matters**: Three independent data sources (overview, weather, crowd), all three use catch{} blocks with no error state, retry, or toast. Users see "暂无数据" and zeros — can't distinguish "backend down" from "no data yet." For a page called "数据大屏," this is trust-destroying.
**Fix**: Add error refs, conditional el-alert with retry buttons, distinguish empty-data from error states.
**File**: `src/views/dashboard/DashboardSmall.vue`
**Suggested command**: `/impeccable harden src/views/dashboard/DashboardSmall.vue`

### [P1] 文化即组件 principle is not delivered
**Why it matters**: Design Principle 1 says "Element Plus是骨架，潮汕文化是血肉." In reality, Element Plus is both. FoodCard and HeritageCard are structurally identical. The only cultural visual element in the entire codebase is the "潮" logo character. Everything else is stock UI with tokenized colors.
**Fix**: Food cards → 陶土-brown borders; heritage cards → 嵌瓷 geometric corner decorations; community cards → 骑楼 arch-shaped headers. Extend the "潮" logomark approach across the UI.
**File**: `src/components/business/FoodCard.vue`, `HeritageCard.vue`
**Suggested command**: `/impeccable bolder src/components/business`

### [P2] No breadcrumb navigation on any detail page
**Why it matters**: FoodDetail, HeritageDetail, PostDetail, TripDetail, PostCreate all lack breadcrumbs. Users from search/external links have zero spatial orientation. Violates Design Principle 3 (双速设计).
**Fix**: Add `el-breadcrumb` to all detail templates: 首页 > [Section] > [Item Name].
**File**: `src/views/food/FoodDetail.vue` (and 4 other detail pages)
**Suggested command**: `/impeccable layout src/views`

### [P2] AI chat is a hardcoded keyword matcher, not real AI
**Why it matters**: Platform positions as "以AI为引擎" but `localReply()` is a 10-entry JS object. Any query outside keywords returns "我还在学习中." When a first-time user asks their third question and hits the wall, trust is breached.
**Fix**: Connect to real LLM backend with RAG, or expand knowledge base to 100+ entries, or be transparent that this is a curated FAQ bot.
**File**: `src/views/CustomerService.vue`
**Suggested command**: `/impeccable harden src/views/CustomerService.vue`

### [P2] --muted color token fails WCAG AA contrast
**Why it matters**: `--muted: oklch(0.55 0.005 248)` yields ~3.3-4.0:1 against white bg, below 4.5:1 AA minimum. Used pervasively (60+ occurrences across 20+ files) for secondary text, timestamps, descriptions, empty states. This is the single most widespread accessibility gap.
**Fix**: Darken `--muted` to at least `oklch(0.45 0.005 248)` or keep L=0.55 but increase chroma to improve readability on white.
**File**: `src/assets/styles/global.css`
**Suggested command**: `/impeccable audit src/assets/styles/global.css`

### [P3] Emoji overload as primary visual language
**Why it matters**: Emojis (🍲🎭🍵👥💬🤖📊🧭) render differently across OS/browser, signal placeholder-quality design, and undermine "文化即组件" by using generic Unicode instead of Chaoshan-specific visual design.
**Fix**: Replace with Element Plus icons + custom SVG. Reserve emoji for playful moments only.
**File**: `src/views/food/FoodList.vue` (and 10+ other files)
**Suggested command**: `/impeccable polish src/views`

## Minor Observations

- SearchBar.vue:131 — `text-transform: uppercase` has no effect on CJK but `letter-spacing: 0.05em` still applies, creating odd typography
- Home.vue:219 — `import { computed }` placed after it's already used (line 209); hoisting works but is disorganized
- DashboardSmall.vue:194 — Shadow `oklch(0 0 0 / 0.03)` is nearly imperceptible on white; cards appear flat
- FoodCard/HeritageCard share identical `formatCount()` — extract to shared utility
- Router scrollBehavior always returns `{ top: 0 }`, losing scroll position on back/forward navigation
- CustomerService mock delay `setTimeout(r, 800 + Math.random() * 1200)` is a nice touch but should be documented
- The "潮" logo character is the single strongest cultural element — this should be the template for all visual decisions
- No `<meta name="theme-color">` or manifest.json for PWA — despite mobile-native mandate

## Questions to Consider

1. If `--primary` flips from blue to 英歌舞朱红 tomorrow, how many components genuinely feel like a Chaoshan culture platform vs. just being recolored Element Plus? Is the gap between "cultural platform" and "CRUD app" truly just a color hex value?

2. The AI chatbot is the headline differentiator ("以AI为引擎"), yet its frontend implementation is a 10-entry JS object. When a first-time user asks their third question and hits the "我还在学习中" wall, have you gained or lost their trust?

3. What if every category icon was a miniature Chaoshan cultural artifact — a tiny 工夫茶杯, an 英歌槌, a 嵌瓷 shard — instead of Element Plus generic icons or Unicode emojis?
